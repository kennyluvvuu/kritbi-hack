# [LE]SBI – Решение для Хакатона: Алгоритм «КАЧИНАТОР» 🌊

Добро пожаловать в репозиторий команды **[LE]SBI**! Наше решение создано для мониторинга и высокоточного прогнозирования уровня воды (паводков) на реке Кача на основе исторических и синтетических данных.

## 🎯 О проекте

Основная цель нашего проекта — предсказывать относительный уровень воды (`water_level_rel`) на горизонтах **6 часов, 24 часа и 72 часа**. Мы разработали комплексную микросервисную архитектуру для мониторинга, симуляции и, самое главное, прогнозирования паводковых ситуаций, а также своевременного оповещения населения.

Наш главный интеллектуальный модуль носит название **Алгоритм «КАЧИНАТОР»**.

---

## 🧠 Алгоритм «КАЧИНАТОР» (Predictor)

В основе нашего решения лежит продвинутая модель машинного обучения на базе **CatBoost**. Мы выбрали этот алгоритм за его способность отлично справляться с табличными данными, устойчивость к переобучению и высокую скорость работы.

### Особенности ML-пайплайна:

- **Основная модель:** CatBoost (для горизонтов 6ч, 24ч, 72ч).
- **Бейзлайн:** Для сравнения качества была реализована базовая модель на основе `LinearRegression`.
- **Feature Engineering (Признаковое пространство):**
  Мы не ограничились только историческими значениями уровня воды. Для повышения точности прогноза в модель добавлены внешние и расчетные данные:
    - **Внешние данные:** Максимальная и минимальная температура (`t_max`, `t_min`), влажность поверхности почвы (`SMsurf`).
    - **Временные лаги:** Исторические значения уровня за последние 7 дней (`lvl_lag_1` ... `lvl_lag_7`).
    - **Скользящие статистики:** Среднее за 7 дней (`lvl_roll_mean_7`), стандартное отклонение за 3 дня (`lvl_roll_std_3`).
    - **Динамика:** Скорость изменения уровня воды (`level_velocity`).
    - **Сезонность и календарь:** День года (`day_of_year`), месяц (`month`), день недели (`day_of_week`).

Весь процесс обучения и инференса инкапсулирован в микросервисе `predictor`, который предоставляет удобное API для получения предсказаний.

## 🤖 AI-Интеграции и Уведомления

В процессе развития проекта мы добавили ряд важных пользовательских функций для повышения интерактивности и безопасности:

- **Телеграм-бот (@KachaAlertBot):** Автоматические уведомления при достижении критических отметок уровня воды. Бот ведет непрерывный мониторинг и предупреждает, если уровень превышает **200 см (Опасность)** или **250 см (Критический уровень)**.
- **AI Чат-бот на базе Ollama LLM (Qwen):** Интегрирован в frontend-виджет прогноза. Чат-бот представлен в виде стилизованного персонажа (**"ПРОГНОЗ ОТ КАЧИНАТОРА(никогда не ошибается)"**) тематики Акинатора. Он предоставляет неформальные консультации о рисках затопления, общается в дружелюбном стиле и отвечает на вопросы о реке Кача.
- **Модуль Симуляции Паводков:** Разработан специальный скрипт для воспроизведения исторических сценариев наводнений (на основе файла `9326_1.csv`). Это позволяет тестировать поведения предиктора и системы уведомлений в стрессовых условиях (например, подача 30-дневной последовательности предшествующей реальному наводнению).

## 🏗 Архитектура решения

Наш проект разбит на несколько изолированных микросервисов:

1.  **Predictor (`/predictor`)**: ML-модуль («КАЧИНАТОР») на Python (CatBoost, Pandas, Scikit-learn), отвечающий за выдачу прогнозов. Запускается как FastAPI-сервис.
2.  **Backend (`/backend`)**: Основной API-сервис на базе Node.js (Bun) с использованием чистой архитектуры. Управляет бизнес-логикой, сохраняет данные в базу (PostgreSQL) и взаимодействует с предикторами.
3.  **Frontend (`/frontend`)**: Пользовательский веб-интерфейс (React + TypeScript + Vite). Включает в себя интерактивные виджеты, визуализацию уровня воды, интеграцию с Telegram и AI-чатом.
4.  **Bot (`/bot`)**: Отдельный сервис Telegram-бота для непрерывного мониторинга и рассылки алертов.
5.  **Simulator (`/simulator`)**: Сервис для генерации (симуляции) потока данных с датчиков мостов в реальном времени, а также для стресс-тестирования системы сценариями паводков.

## 🚀 Как запустить проект

Мы подготовили Docker-окружение для легкого развертывания всего комплекса в один клик.

### Предварительные требования

- Установленный [Docker](https://docs.docker.com/get-docker/) и [Docker Compose](https://docs.docker.com/compose/install/).

### Шаги запуска:

1. **Клонируйте репозиторий:**

    ```bash
    git clone <url_вашего_репозитория>
    cd <название_папки>
    ```

2. **Настройте переменные окружения:**
   Скопируйте пример файла конфигурации:

    ```bash
    cp .env.example .env
    ```

    _(В `.env` необходимо указать ваши токены для Telegram-бота, ключи базы данных и т.д.)._

3. **Запустите контейнеры:**

    ```bash
    docker-compose up -d --build
    ```

    Если вы не хотите запускать симулятор, можно использовать `docker-compose -f docker-compose.no-sim.yml up -d --build`.

4. **Доступ к сервисам:**
    - **Frontend (UI):** [http://localhost:5173](http://localhost:5173)
    - **Backend API:** [http://localhost:3000](http://localhost:3000)
    - **Predictor API:** [http://localhost:8000](http://localhost:8000)

---
